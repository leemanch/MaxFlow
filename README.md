# University Bot Management System

Бот для управления университетскими процессами с различными ролями пользователей + мини-приложение с расписанием.

## Функциональность
### Менеджер админов
- Возможность добавить/удалить админов
- Возможность просмотреть список админов
- Проверить статус пользователя

### **Администратор**
- Управление ролями пользователей (выдача/снятие)
- Черный список пользователей (добавление/удаление)
- Модерация заявок на разбан
- Просмотр заявок от деканата
- Статистика и мониторинг системы

### **Сотрудник деканата**
- Обработка заявок студентов на справки об обучении
- Просмотр и управление заявками
- Одобрение/отклонение запросов

### **Студент**
- Заказ справок об обучении - автоматизированная подача заявок
- Подписки на новости - ВУЗа и общежития
- Жалобы и проблемы - сообщения о проблемах в общежитии
- Запросы на пропуск - восстановление пропусков
- Электронная библиотека - информация о доступе

### **Абитуриент**
- Информация о ВУЗе - история, интересные факты
- События - предстоящие мероприятия и дни открытых дверей

### **SMM**
- Управление новостями - добавление, редактирование, удаление
- Управление событиями - создание и редактирование мероприятий
- Рассылка информации - массовые уведомления

### **Заведующий общежитием**
- Жалобы студентов - обработка проблем в общежитии
- Заявки на пропуск - управление пропусками
- Рассылка информации - уведомления для жильцов

## **Технические особенности**
- SQLite - легковесное хранение данных
- Автоматическое создание таблиц - при первом запуске

### Роли пользователей:
- **Студент** - заказ справок, жалобы, запросы пропусков
- **Абитуриент** - информация о ВУЗе, события
- **Деканат** - обработка заявок студентов
- **Администратор** - управление ролями, черный список
- **SMM** - управление новостями и событиями
- **Заведующий общежитием** - жалобы студентов, пропуски

### Основные возможности чат-бота:
- Система заявок (справки об обучении, пропуски, жалобы)
- Черный список пользователей
- Рассылка новостей
- Управление событиями
- Модерация заявок

### Основные команды бота
- /start - начать работу, выбор роли
- /menu - главное меню (доступно после выбора роли)
- /setd - подать заявку на роль деканата
- /unban_request - заявка на разбан из ЧС
- /getadmin - получить права администратора (для тестирования)

### Основные возможности мини-приложения:
- Расписание по группам
- Расписание преподавателей
- Выбор подгруппы
- Выбор избранной группы
- Поиск по преподавателям или подгруппе

## Требования к запуску
- Установленный Python 3.11
- Установленный Docker Desktop(опционально)

## Локальный запуск(с использованием Docker)

1. Клонируйте репозиторий
```bash
git clone https://github.com/leemanch/MaxFlow.git
cd MaxFlow
```
2. Откройте консоль (ctrl + `)
3. Соберите образ 
```commandline
docker build -t university-bot .
```
4. Запустите контейнер
```commandline
docker run -d --name university-bot \
  -e BOT_TOKEN="bot_token_here" \
  university-bot
```

## Локальный запуск на Windows(без Docker)
1. Клонируйте репозиторий
```bash
git clone https://github.com/leemanch/MaxFlow.git
cd MaxFlow
```
2. Откройте консоль (ctrl + `)
```python
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
echo "BOT_TOKEN=your_bot_token_here" > .env
python main.py
```
3. В новой консоли выполнить команду
```python
venv\Scripts\activate
python admin_manager.py
```
4. Выдать админа в боте для тестирования с помощью консольного интерфейса
